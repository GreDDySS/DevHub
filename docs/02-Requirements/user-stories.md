# User Stories

> Формат: *Как [роль], я хочу [действие], чтобы [цель]*

---

## Epic 1 — Управление проектами

### US-001: Добавление проекта

> **Как** разработчик, **я хочу** добавить проект в каталог, **чтобы** видеть его в общем списке.

**Acceptance Criteria:**
- [ ] Кнопка «Добавить проект» доступна на главном экране
- [ ] Форма содержит поля: название (обязательное) или Название от папки, путь к папке (обязательное), язык, описание
- [ ] Путь выбирается через диалог выбора папки
- [ ] После сохранения проект появляется в списке
- [ ] При отсутствии обязательных полей — показывается ошибка

**Priority:** Must | **Sprint:** MVP

---

### US-002: Просмотр списка проектов

> **Как** разработчик, **я хочу** видеть все свои проекты в одном списке, **чтобы** быстро находить нужный.

**Acceptance Criteria:**
- [ ] Проекты отображаются в виде карточек/списка
- [ ] Каждая карточка показывает: название, язык, статус, дату последнего изменения
- [ ] Список можно сортировать по имени / дате / статусу
- [ ] Список можно фильтровать по статусу (Active / Completed / Archived)
- [ ] Поиск по названию работает в реальном времени
- [ ] Можно поменять режима отображение Карточки/список

**Priority:** Must | **Sprint:** MVP

---

### US-003: Открытие папки в Explorer

> **Как** разработчик, **я хочу** открыть папку проекта в Проводнике одной кнопкой, **чтобы** не искать её вручную.

**Acceptance Criteria:**
- [ ] На каждой карточке проекта есть кнопка «Открыть папку»
- [ ] При нажатии открывается Windows Explorer с выбранной папкой
- [ ] Если путь не существует — показывается предупреждение

**Priority:** Must | **Sprint:** MVP

---

### US-004: Запуск проекта в IDE

> **Как** разработчик, **я хочу** открыть проект в нужной IDE одним кликом, **чтобы** начать работу быстрее.

**Acceptance Criteria:**
- [ ] На карточке проекта есть кнопка «Открыть в IDE»
- [ ] В настройках можно указать пути к VS Code, Visual Studio, Rider
- [ ] IDE по умолчанию выбирается автоматически по языку проекта (опционально)
- [ ] Если IDE не настроена — показывается уведомление с ссылкой на настройки

**Priority:** Must | **Sprint:** MVP

---

### US-005: Редактирование проекта

> **Как** разработчик, **я хочу** редактировать информацию о проекте, **чтобы** поддерживать данные актуальными.

**Acceptance Criteria:**
- [ ] Двойной клик или кнопка «Редактировать» открывает форму
- [ ] Форма предзаполнена текущими данными
- [ ] Сохранение обновляет данные в списке и JSON-файле

**Priority:** Must | **Sprint:** MVP

---

### US-006: Удаление проекта

> **Как** разработчик, **я хочу** удалить проект из каталога, **чтобы** не захламлять список.

**Acceptance Criteria:**
- [ ] Кнопка «Удалить» с диалогом подтверждения
- [ ] Удаление убирает проект из списка и JSON-файла
- [ ] Файлы проекта на диске НЕ удаляются

**Priority:** Must | **Sprint:** MVP

---

## Epic 2 — Настройки

### US-007: Настройка IDE

> **Как** разработчик, **я хочу** настроить пути к моим IDE, **чтобы** открывать проекты в нужной среде.

**Acceptance Criteria:**
- [ ] Страница настроек содержит секцию «IDE»
- [ ] Можно указать путь к VS Code, Visual Studio, Rider
- [ ] Каждый путь валидируется (файл существует)
- [ ] Настройки сохраняются в `settings.json`

**Priority:** Must | **Sprint:** MVP

---

## Epic 3 — Захват ссылок (Phase 2)

### US-008: Захват ссылки горячей клавишей

> **Как** разработчик, **я хочу** сохранять ссылку нажатием Ctrl+Shift+Y, **чтобы** не потерять полезный ресурс.

**Acceptance Criteria:**
- [ ] Глобальная горячая клавиша работает из любого приложения
- [ ] Ссылка из буфера обмена автоматически сохраняется
- [ ] Появляется уведомление о сохранении (toast notification)
- [ ] Ссылка появляется в разделе «Ссылки» приложения

**Priority:** Should | **Sprint:** Phase 2

---

### US-009: Просмотр сохранённых ссылок

> **Как** разработчик, **я хочу** просматривать все сохранённые ссылки, **чтобы** находить нужные ресурсы.

**Acceptance Criteria:**
- [ ] Раздел «Ссылки» со списком всех сохранённых URL
- [ ] Каждая ссылка показывает: URL, дату сохранения
- [ ] Поиск по URL и тегам

**Priority:** Should | **Sprint:** Phase 2

---

## Epic 4 — Автозапуск и трей (Phase 2)

### US-010: Сворачивание в трей

> **Как** разработчик, **я хочу** чтобы приложение сворачивалось в трей при закрытии, **чтобы** оно всегда было под рукой.

**Acceptance Criteria:**
- [ ] При нажатии «Закрыть» окно сворачивается в системный трей
- [ ] Иконка в трее — клик открывает окно
- [ ] Контекстное меню трей-иконки: Открыть / Выход
- [ ] В настройках можно отключить сворачивание в трей

**Priority:** Should | **Sprint:** Phase 2

---
