# Non-Functional Requirements

---

## NFR-0xx — Производительность

| ID | Требование | Критерий | Приоритет |
|----|------------|----------|-----------|
| NFR-001 | Время запуска | Приложение запускается менее чем за **2 секунды** | **Must** |
| NFR-002 | Время отклика UI | Все взаимодействия с интерфейсом — менее **100 мс** | **Must** |
| NFR-003 | Объём данных | Приложение корректно работает с **1000+ проектов** без деградации | **Should** |
| NFR-004 | Потребление RAM | Рабочее потребление памяти — менее **150 МБ** | **Should** |
| NFR-005 | Размер установщика | Установочный файл — менее **50 МБ** | **Could** |

## NFR-1xx — Надёжность

| ID | Требование | Критерий | Приоритет |
|----|------------|----------|-----------|
| NFR-101 | Сохранность данных | JSON-файлы не повреждаются при аварийном завершении | **Must** |
| NFR-102 | Автосохранение | Данные сохраняются автоматически после каждого изменения | **Must** |
| NFR-103 | Восстановление | При повреждении файла — уведомление и предложение восстановления из бэкапа | **Should** |
| NFR-104 | Обработка ошибок | Ни одна ошибка не приводит к падению приложения (graceful degradation) | **Must** |

## NFR-2xx — Безопасность

| ID | Требование | Критерий | Приоритет |
|----|------------|----------|-----------|
| NFR-201 | Локальность данных | Все данные хранятся только локально, без отправки в сеть | **Must** |
| NFR-202 | Права доступа | Приложение не требует прав администратора | **Must** |
| NFR-203 | Безопасные пути | Валидация путей для предотвращения path traversal | **Should** |

## NFR-3xx — Usability

| ID | Требование | Критерий | Приоритет |
|----|------------|----------|-----------|
| NFR-301 | Обучение | Новый пользователь может добавить первый проект за **30 секунд** | **Must** |
| NFR-302 | Клавиатурная навигация | Все основные действия доступны с клавиатуры | **Should** |
| NFR-303 | Accessibility | Интерфейс совместим с screen reader (UI Automation) | **Could** |
| NFR-304 | Масштабирование UI | Интерфейс корректно отображается при масштабе 100-150% | **Should** |

## NFR-4xx — Поддерживаемость

| ID | Требование | Критерий | Приоритет |
|----|------------|----------|-----------|
| NFR-401 | Архитектура | Чёткое разделение на слои (MVVM + Clean Architecture) | **Must** |
| NFR-402 | Покрытие тестами | Бизнес-логика покрыта unit-тестами на **80%+** | **Should** |
| NFR-403 | Документация | Каждый публичный класс/метод задокументирован через XML comments | **Should** |
| NFR-404 | Логирование | Структурированное логирование всех ключевых операций | **Should** |

## NFR-5xx — Совместимость

| ID | Требование | Критерий | Приоритет |
|----|------------|----------|-----------|
| NFR-501 | ОС | Windows 10 (1809+) и Windows 11 | **Must** |
| NFR-502 | .NET | .NET 10 (LTS) | **Must** |
| NFR-503 | Архитектура | x64, поддержка ARM64 через эмуляцию | **Should** |
